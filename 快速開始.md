## 🎯 下一步：啟動後端專案

### 步驟 1: 確保環境已安裝

```bash
# 檢查 Java 版本（需要 17+）
java -version

# 檢查 Maven 版本（需要 3.8+）
mvn -version

# 檢查 MySQL 是否啟動
mysql --version
```

### 步驟 2: 建立資料庫

```bash
cd "C:\史晨\商研院上課內容\CryptoDashboard"
mysql -u root -p < database/schema.sql
```

輸入 MySQL 密碼後，資料庫 `crypto_dashboard` 將被建立。

### 步驟 3: 配置資料庫連線

編輯 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    password: your_mysql_password  # 修改這裡！
```

### 步驟 4: 啟動後端

```bash
cd backend
mvn spring-boot:run
```

**成功啟動後會看到：**

```
========================================
CryptoDashboard Backend 啟動成功！
API 位址: http://localhost:8080/api
========================================
```

### 步驟 5: 測試後端是否正常

在瀏覽器訪問：
- http://localhost:8080/api （應該顯示 404，因為還沒實作任何 Controller）

這是正常的！接下來需要實作 Repository、Service 和 Controller。

## 🐛 常見問題

### Q: Maven 下載依賴很慢？
A: 設定使用國內 Maven 鏡像（阿里雲）：
```xml
<!-- 編輯 ~/.m2/settings.xml -->
<mirror>
  <id>aliyun</id>
  <url>https://maven.aliyun.com/repository/public</url>
  <mirrorOf>central</mirrorOf>
</mirror>
```

### Q: 啟動時出現 "Table 'users' doesn't exist"？
A: 確保已執行 `mysql -u root -p < database/schema.sql`

### Q: 啟動時出現資料庫連線錯誤？
A: 檢查：
1. MySQL 是否啟動
2. `application.yml` 中的密碼是否正確
3. 資料庫 `crypto_dashboard` 是否已建立

### Q: 如何查看 SQL 查詢？
A: 已在 `application.yml` 設定 `show-sql: true`，SQL 會顯示在控制台。
